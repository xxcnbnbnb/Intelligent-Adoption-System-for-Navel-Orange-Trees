## 架构设计原则

1. **职责分离**：每层专注于特定职责
2. **依赖倒置**：上层依赖抽象接口
3. **单一职责**：每个模块只负责一个功能
4. **开放封闭**：对扩展开放，对修改封闭
5. **接口隔离**：客户端不应依赖不需要的接口

## 五层架构

### Middleware 层
- 身份认证和授权
- 请求日志记录
- 错误处理
- CORS跨域处理
- 请求限流
- 参数预处理

### Controller 层
- 接收HTTP请求
- 参数验证和转换
- 调用Service层处理业务逻辑
- 格式化响应数据
- 处理HTTP状态码

### Service 层
- 实现核心业务逻辑
- 管理数据库事务
- 协调多个Repository的数据访问
- 数据关联验证
- 业务规则校验
- 调用外部服务

### Repository 层
- 封装数据库操作
- 构建复杂查询
- 实现数据缓存策略
- 提供数据访问接口
- 处理数据持久化

### Model 层
- 定义数据模型结构
- 配置表关系映射
- 数据验证规则
- 虚拟字段定义
- 钩子函数配置
